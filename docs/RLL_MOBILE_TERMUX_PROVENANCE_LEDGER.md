# RLL Mobile / Termux Provenance Ledger

Status: provenance ledger with strict evidence separation.

Purpose: document the claim that early RLL calculations, figures and data-station artifacts were produced from a mobile/Termux-style environment, while separating verified repository evidence from still-unverified operational provenance.

## 1. Executive status

The `v1.0.0` README verifies that RLL already had:

- a named model;
- a Friedmann-extension formula;
- `Ω_s0` and `f(a)`;
- DE→matter superposition language;
- magnetic/plasma terms;
- observable directions;
- references to figures/CSVs in the README text.

However, a repository search for direct terms such as `Termux`, `celular`, `Android`, `proot`, `mobile phone` and related keywords did not return direct evidence in the current indexed search.

Therefore:

- mobile/Termux execution is **not denied**;
- mobile/Termux execution is **not yet verified from repository text**;
- it remains a provenance claim requiring artifacts, logs, screenshots, file metadata, or scripts.

## 4. What counts as proof for mobile/Termux provenance

Acceptable evidence:

1. Terminal logs showing commands and device prompt.
2. Shell scripts with paths typical of Termux, such as `/data/data/com.termux/files/home`.
3. `uname -a`, `getprop`, `termux-info`, `python --version`, `pip freeze` logs.
4. File metadata from original generated images/CSVs.
5. Screenshot sequence showing the calculation run and output generation.
6. Git commits containing generated artifacts and their timestamps.
7. A manifest linking command → input → output → hash.
8. ZIP/DOI archive containing scripts and outputs from the mobile environment.

## 7. Correct language
