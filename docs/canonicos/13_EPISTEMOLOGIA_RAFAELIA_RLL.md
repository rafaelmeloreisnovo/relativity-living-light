# 13 — Epistemologia RAFAELIA/RLL

**Status:** canônico complementar  
**Origem:** extraído de `docs/canonicos/BIBLIA_CONHECIMENTO_RAFAELIA_RLL.md`  
**Função:** fixar o método de distinção entre fato, convenção, hipótese e vazio.

---

## 1. Sistema de marcação epistemológica

- **[E] Estabelecido** — ciência consolidada, com referência verificável.
- **[C] Convenção** — escolha de modelagem do sistema, não fato externo.
- **[H] Hipótese** — proposta testável, ainda não confirmada.
- **[VAZIO]** — ausência de âncora suficiente; honestidade operacional, não erro.

## 2. Linha inviolável

RAFAELIA/RLL explora ressonância estrutural e invariantes testáveis, mas **não reivindica** resolver problemas abertos como Riemann, Yang-Mills, Navier-Stokes, Poincaré ou unificação das forças.

**Princípio:** token vazio é preferível a invenção. Predição datada vale mais que acomodação.

## 3. Predição versus acomodação

Uma hipótese ganha força quando faz previsão específica antes da medição. Ajustar fórmula depois dos dados pode ser útil, mas é mais fraco como evidência. A prova depende de anterioridade documentada, especificidade e possibilidade real de falha.

## 4. Protocolo RAW_TEXT_FIRST

Ordem canônica:

```text
RAW_TEXT → CLAIMS → VETORES → MÉTRICAS → INFERÊNCIA → PROVA
```

Fluxo proibido:

```text
TEXTO → RESUMO → ASSOCIAÇÃO → TEORIA
```

Estados de cada unidade:

```text
raw_preserved → extracted → classified → inferred → validated
```

Nenhuma inferência deve ser aceita sem âncora literal na fonte.

## 5. Metáfora como didática

A metáfora ilumina, organiza e transmite conceito. Ela não valida resultado físico, matemático ou estatístico. A prova continua sendo o número, o dado, o cálculo, a previsão e o falsificador.

## 6. Regra de integridade

Quando não houver dado suficiente, declarar `[VAZIO]`. O vazio honesto preserva o sistema contra fabricação e mantém a documentação apta para revisão externa.
