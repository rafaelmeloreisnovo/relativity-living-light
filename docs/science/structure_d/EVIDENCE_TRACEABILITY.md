# Structure D — Evidence Traceability

Status: canonical traceability matrix  
Claim: `claim_allowed=false`

## Rule

No result claim should enter a paper, README, report or presentation unless the chain below is explicit:

```text
source file -> columns/keys -> transformation -> artifact path -> status
```

## Gate checklist

```text
[ ] Claim has source path.
[ ] Source columns/keys are named.
[ ] Transformation is reproducible.
[ ] Artifact path is concrete.
[ ] Status is one of: Supported / In validation / Hypothesis / Blocked.
[ ] Claim boundary is explicit.
```

## Safe conclusion

Traceability is the bridge between historical notes and publishable claims. Without an artifact path and a baseline, a sentence remains hypothesis or blocked claim.
