1. Todo conteúdo deve manter a integridade simbiótica e científica do projeto.  
2. Pull requests e issues devem respeitar o Manifesto e a Licença Viva.  
3. Alterações maiores exigem consenso explícito e documentação multilíngue.

5. Este repositório é vivo: ciência, espiritualidade e ética coexistem.

## Convenção única de naming (documentação Markdown)

**Padrão oficial:** `UPPER_SNAKE_CASE.md`

Regras obrigatórias para novos arquivos em `docs/`:
- usar apenas letras maiúsculas (`A-Z`), números (`0-9`) e `_` no nome base;
- não usar espaços;
- não usar pontuação duplicada (ex.: `..`, `__`, `--`);
- manter sufixo único `.md`.

Exemplos válidos: `RELEASE_NOTES_HISTORY.md`, `DOCUMENTATION_FULL_INVENTORY.md`.
Exemplos inválidos: `Easy..md`, `New theory and beyond.md`, `Readme.md`.
