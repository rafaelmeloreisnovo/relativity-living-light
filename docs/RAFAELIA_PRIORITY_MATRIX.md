# RAFAELIA · Matriz de prioridade vetorial para Relativity Living Light

Objetivo: proteger o eixo científico do projeto, separando hipótese, formulação, simulação, comparação observacional e prova.

## Vetor base

```text
v = (intencao, observacao, ruido, transmutacao, memoria, coerencia, urgencia, latencia, evidencia, obviedade)
```

## Peso tensorial

```text
valor = 0.20*evidencia + 0.18*urgencia + 0.16*memoria + 0.14*transmutacao + 0.12*latencia + 0.08*obviedade + 0.06*intencao + 0.04*ruido + 0.02*coerencia
```

## Prioridades científicas

| Camada | Aplicação | Critério de prova |
|---|---|---|
| Evidência | dados, simulações, comparação com literatura e observáveis | fonte, equação e reprodução mínima |
| Urgência | afirmação forte sem validação | rebaixar para hipótese ou validar |
| Obviedade | definições, unidades, domínio de validade | variável definida antes de uso |
| Latência | fórmulas soltas, intuições geométricas, imagens conceituais | converter em definição ou hipótese |
| Memória | ligação com Cosmos, Catalogo-cosmologico, Fisica e RAFAELIA | mapa de integração sem prova inflada |

## Estados científicos

```text
[DEFINICAO]     termo/variável/domínio bem definido
[HIPOTESE]      proposta coerente ainda sem teste suficiente
[SIMULACAO]     resultado numérico com código/dados
[COMPARACAO]    confrontado com modelo/dado externo
[REFUTADO]      falhou em teste claro
[VALIDADO]      passou em teste delimitado, não universal
```

## Blending correto

| Tipo | Tratamento |
|---|---|
| fórmula | definir variáveis, unidades e domínio |
| cosmologia | comparar com ΛCDM/observáveis quando aplicável |
| gráfico/figura | registrar origem, método e interpretação limitada |
| texto filosófico | manter como motivação, não como prova física |
| simulação | salvar código, parâmetros, seed e saída |
